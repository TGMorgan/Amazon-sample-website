//Advanced functions//
    1.setTimeout()

        It allows us to run a function in the future.
        It takes two parameters. The first is a function that we want to run in the future
        The second parameter is how long you want to wait before running this function.(a number in milliseconds)
        NB: 1000 milliseconds = 1 second.
        We give it a function and it calls the function after a certain amount of time by itself.
        It's an Asynchronous code. It sets the timer in the background and then goes to the next line of code.

        Asynchronous code doesn't wait for the code line to finish executing before going to the next line.

        Synchronous code is a type of code that the computer have to wait for it to finish executing before going to the next line.

        NB: It always returns an ID that can be used to stop it ie. clearTimeout()

    2. setInterval()

        It takes the same two parameters as setTimeout().
        The first one is a function that we want to run in the future.
        It allows us to run function in the future just like setTimeout. But this time, it keeps running the function after a certain period of time.

        NB: It always returns an ID that can be used to stop the loop.
        clearInterval() is used to stop an interval.

    3. .forEach() method

        It's another way to loop through an array.
        Syntax:
        let array = [];

        array.forEach(function(value, index){
            //Code to be executed.
        });

        if you want to use "continue" to skip over an iteration in the .forEach() method, use "return"

        There is no other way to use break though. So if you need to use break, then it's better to use a regular for loop first.

    4. Arrow function.
        It works mostly like a regular function.

        //Syntax
        let arrowFunction = () =>{
            console.log();
        }
        And you can call it by "arrowFunction();"

        When an arrow function has one parameter, the round brackets are actually optional.

    5. .addEventListener();
        It let's us run some code when we interact with the element.

        It takes in two parameters.
        The first is the event (What type of interaction we want to listen for.) NB: Eg. onclick is click.
        The second is a function we want to run.
        You can remove add event listener using a method called ".removeEventListener()"

    6.filter() method.

        It's an array method.
        It works the same as .forEach().

        //What it does?//

        1. It creates a new array.
        2. If inner function returns true it will put the value in the array.
        3. If inner function returns false, it will not put the value in the array.

    7. .map() method
        It's an array method.
        It works the same as .forEach() and .filter()

        //What it does?//

        1. It creates a new array.
        2. Whatever we return, will be added to the new array.

In Jasmine, to create a tests suite, we're going to use a function called: describe()

But in Object-Oriented Programming, we organize all of the code into an object.

2. It's easy to create multiple objects.

//Class//

A class is basically an object generator.
It makes it easier to create objects.
Inside a class, we are gonna put the properties and methods we want for each object that we generate.

When calling a class, for rg. class mobile(), it's called bu using   "new mobile()".

Each object that we generate from a class is called an Instance of class.

[object] instanceof [className]      (It checks if an object is an instance of a class).

NB: Object oriented programming means organizing our code into objects because it tries to represents the real word
And a class is a feature that helps us to generate these objects.

Benefits of classes
1. A class looks like the object that we generate.

2. They have extra features for object oriented programming.

    * Constructor method
      This let's us run some setup code after creating an object.
      When we generate an object(instance of a class), it will run the constructor automatically.

      Constructors can also have parameters.

      A value is saved into the parameter by placing the value into the part of your code where you created the instance of the class.
      Eg. let cart = new Cart([place value here to assign value to constructor parameter])

      NB: * The method must be named "constructor". You cannot use any name that you want.
          * Do not return anything from a constructor.

    * Private properties and method
      Private - This means a property can only be accessed inside the class.
      This is applied by the properties by adding the character '#' in-front of the property in the class.
      And also when ever that property is accessed inside the class, '#' is added in-front.

      It prevents us from accessing a property outside the object generated from a class and messing it up.

      Public- They are properties without a '#' in-front, meaning it can be accessed everywhere.

      NB: field is the same as a property.

      //Private methods//
      To make a method private, we do the same thing. A '#' is placed in-front of the method.
