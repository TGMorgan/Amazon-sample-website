    //Objects//

    Objects allows us to group multiple values together and then allows
    us to access or change these values using the properties in it.

    Syntax for objects =

    Eg. const product = {
        name: 'Socks',
        price: 199.99
    };

    NB: Property pairs are separated with a comma.

    To access a value inside the object:

    product.name(dot notation)

    OR

    product[name](bracket notation).
    NB: Bracket let's us use properties that normally don't work with dot notation

    NB: If you access a property that doesn't exist, the value will be undefined.

//Advanced functions//
    1.setTimeout()

        It allows us to run a function in the future.
        It takes two parameters. The first is a function that we want to run in the future
        The second parameter is how long you want to wait before running this function.(a number in milliseconds)
        NB: 1000 milliseconds = 1 second.
        We give it a function and it calls the function after a certain amount of time by itself.
        It's an Asynchronous code. It sets the timer in the background and then goes to the next line of code.

        Asynchronous code doesn't wait for the code line to finish executing before going to the next line.

        Synchronous code is a type of code that the computer have to wait for it to finish executing before going to the next line.

        NB: It always returns an ID that can be used to stop it ie. clearTimeout()

    2. setInterval()

        It takes the same two parameters as setTimeout().
        The first one is a function that we want to run in the future.
        It allows us to run function in the future just like setTimeout. But this time, it keeps running the function after a certain period of time.

        NB: It always returns an ID that can be used to stop the loop.
        clearInterval() is used to stop an interval.

    3. .forEach() method

        It's another way to loop through an array.
        Syntax:
        let array = [];

        array.forEach(function(value, index){
            //Code to be executed.
        });

        if you want to use "continue" to skip over an iteration in the .forEach() method, use "return"

        There is no other way to use break though. So if you need to use break, then it's better to use a regular for loop first.

    4. Arrow function.
        It works mostly like a regular function.

        //Syntax
        let arrowFunction = () =>{
            console.log();
        }
        And you can call it by "arrowFunction();"

        When an arrow function has one parameter, the round brackets are actually optional.

    5. .addEventListener();
        It let's us run some code when we interact with the element.

        It takes in two parameters.
        The first is the event (What type of interaction we want to listen for.) NB: Eg. onclick is click.
        The second is a function we want to run.
        You can remove add event listener using a method called ".removeEventListener()"

    6.filter() method.

        It's an array method.
        It works the same as .forEach().

        //What it does?//

        1. It creates a new array.
        2. If inner function returns true it will put the value in the array.
        3. If inner function returns false, it will not put the value in the array.

    7. .map() method
        It's an array method.
        It works the same as .forEach() and .filter()

        //What it does?//

        1. It creates a new array.
        2. Whatever we return, will be added to the new array.






//HTML Data attribute//
It's just another html attribute.
The purpose of a data attribute is that it allows us to attach any information to an HTML element.

Syntax rules:
A data attribute is just an HTML attribute. That means it follows the same syntax rules.

name = value
eg.
data-product-name = "${product.name}"

NB: 1.The one difference is that data attributes has to start with "data-".
    2. Also make sure to separate the words with a dash ( - ). Also known as kebab case.


.dataset = It gives us all the data attributes attached to an element.
It basically becomes an object.

So calling it will be like: (.dataset.productSize.)
NB: Calling the stored dataset in called using camelCase not kebab-case.




//Modules//

A module contains a variable inside a file. So now, this variable is not going to cause conflicts
with anything outside the file.

To create a module,
1. Create a file and write our code in it.
2. Don't load the file with a script tag.

//How to get a variable out of a file//
1. First, add type = "module" attribute to enable this feature
2. Export  the variables that we want to get out.
3. Import the variables when we need them.

//Notes this about modules//
1. We have to put all of our imports at the top of the file.
2. We need to use live server.

//Benefits of modules//
1. It helps us avoid naming conflicts.



//Syntax//
add type = "module" to main javascript file

export(Your code)

import {variable} from '[file path]';
